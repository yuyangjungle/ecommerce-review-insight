# 电商评论洞察助手项目说明

## 1. 项目概述

### 项目名称

电商评论洞察助手

### 一句话介绍

帮助电商运营或商品经理从非结构化评论中快速提炼用户痛点、卖点机会、FAQ 草案与优化方向，并通过引用与提示词对比提升结果可解释性。

### 当前阶段

当前为：

`可交互演示版 / MVP 验证版`

## 2. 背景问题

在商品运营工作中，评论与问答是最重要的一手用户反馈来源，但它们存在三个典型问题：

1. 信息量大，人工通读成本高。
2. 表达分散，难以快速形成结构化判断。
3. AI 总结虽然快，但容易出现“看起来像对的，实际上没有依据”的问题。

因此，这个项目关注的不只是“能不能生成结论”，而是：

- 能不能把评论转成可执行的产品结论
- 能不能让结论可回溯、可验证
- 能不能比较不同工作流的质量差异

## 3. 目标用户

主要用户：

- 电商运营
- 商品经理
- 内容运营

次级用户：

- AI 产品经理
- 算法策略同学

## 4. 产品目标

这个项目希望帮助用户在短时间内完成以下任务：

1. 看懂评论里最重要的正负向主题。
2. 提炼用户最关心的问题。
3. 快速生成可用的卖点建议与 FAQ 草案。
4. 查看结论的原始依据，降低“AI 胡编”风险。
5. 对比不同提示词工作流的效果。

## 5. 方案设计

### 核心思路

项目没有采用泛泛的聊天式交互，而是拆成一条更结构化的工作流：

1. 输入评论数据
2. 抽取主题
3. 提炼问题
4. 生成卖点与 FAQ
5. 附上引用
6. 对比工作流输出质量

### 为什么这么拆

- 先抽主题，再生成内容，更容易控制输出结构。
- 加引用回看，可以提升结果可信度与可解释性。
- 做工作流对比，而不是只给单一输出，有助于展示产品评估意识。

## 6. 当前版本实现了什么

### 前端

- 交互式分析页
- 核心摘要
- 原始评论样本区
- 主题、FAQ、卖点、优化方向卡片
- 提示词 `v1` / `v2` 对比区
- 引用抽屉

### 后端

- 本地 API 服务
- 样例数据读取
- 上传 JSON 数据分析
- 轻量规则抽取与模拟工作流
- DeepSeek / OpenAI 兼容接口的可选接入
- 分析结果结构化返回

### 评估

当前版本展示了轻量评估维度：

- 引用覆盖率
- 结构完整度
- 可用性评分
- 可信度评分

## 7. 关键设计取舍

1. 先抽取主题，再生成内容，能够降低输出结构失控的概率。
2. 为每条结论保留引用，能够让结果具备回溯能力，减少“只有结论、没有依据”的问题。
3. 同时保留 `v1` 与 `v2` 两套工作流，有助于比较不同分析链路的结构完整度与引用覆盖率。
4. 将结果拆为主题、问题、卖点、FAQ 和优化方向几个区域，便于运营场景下的直接消费。

## 8. 当前局限

当前版本的主要局限有：

1. 当前公开页仍是静态展示版，完整后端版尚未长期稳定上线。
2. 真实 LLM 能力已支持接入，但仍需要补更多真实数据和系统化评测。
3. 还没有真实用户反馈与系统化人工评测数据。
4. 上传格式目前仍偏工程化，不够消费级。
